Instrumenting Database Visibility in AppDynamics ï¿½ Process

1.       Get list of production and non-production database servers from app team / OR Bill Edwards Database and validate with app SME. (BE SURE TO GET PORT and SERVICE NAME INFORMATION FROM APP TEAM)

2.       Fill out the firewall questionnaire (included in this folder) with server information corresponding to the environment this request is for. (Always use the VIP as the source when possible)

3.       Create ClickIT database user account tickets to create the required user on those boxes for our database collector to log in with. (One for production and one for non-production)

a.      Go to ClickIT > Home, search for database account, and click Database Access, Password Reset, Account Unlock

b.      Click ClickIT Ticket at the top

c.       Under Request Type select New Account

d.      In the Target User IDs: Type app_7b3_user

e.       Under Request Type Select SystemID/Service Account

f.        In the Application ID associated with dbs put in the app name and select the desired environment this account is going to be made for, then click the square next to the field and select the app.

g.       Then click Next

h.      Under the Database(s) for which access is needed: field, click the square and select the databases given from the app team (step 1)

                                                               i.      If there are databases needed not in this list add them in the Databases for which access is needed section directly below

i.         Select Read Only from the Access Needed drop down.

j.        Fill in the Business Justification section with this info: AppDynamics Database Monitoring for Production *APP ID* databases. This request will open connectivity from AppDynamics Database Controller to individual Oracle Databases for read-only access to basic database health metrics.

k.       Fill in the Additional Info section with this info:

ï¿½ CREATE SESSION

ï¿½ SELECT_CATALOG_ROLE

CREATE USER app_7b3_user IDENTIFIED BY password

default tablespace users

temporary tablespace temp;

l.         Click Finish, and take note of the ticket number

m.    Wait for this account to be created and watch for an email from Network Security with a temporary password

ï¿½ï¿½ï¿½ï¿½ï¿½ï¿½ **This will take 1 ï¿½ 2 weeks (step 3 and 4 can be done simultaneously) **

4.       Create ClickIT firewall tickets to enable connectivity from our database visibility servers to the database servers provided in step 1.

a.       Go to ClickIT > Home, search for firewall requests, then click engineering requests. Then click ClickIT Ticket at the top.

ï¿½

b.      Select Firewall Requests from the Request Type drop down box. Then click next.

c.       Fill out the 4 drop down lists accordingly (most of the time all 4 will be set to NO)

d.      Select IT Firewall from the Network or Firewall drop down menu.

e.      Click Add Files button in the middle of the page, the navigate to your filled-out firewall questionnaire from step 2.

f.        Fill out the Overview Description of Request box with this information: ï¿½App dynamics database visibility firewall request to collect metrics on *APP NAME* databases.ï¿½

g.       Fill out the Type of Data Transmitting box with this information: ï¿½Operational performance metrics regarding the Oracle database.ï¿½

h.      Click finish at the top of the page and take note of the ticket number provided.

**This step will take 1 ï¿½ 2 weeks**

5.       Once the firewall request is completed, test the connectivity.

a.       Open putty and enter end our database visibility host (trelk024.test.sprint.com for non-prod and plslj764.corp.sprint.com for production)

b.      Enter credentials to login.

c.       Once logged in type telnet *ip/hostname* *port*. If the return message says connected, then proceed with the next steps.

                                                               i.      If it fails to connect, contact network security for troubleshooting purposes. (Russ Pickerell or Pratik Patel)

6.       Set up the database collector in AppDynamics

a.       Navigate to the AppDynamics user interface

b.      Click on Databases at the top of the screen

c.       Click Add

d.      Select the type of database from the first drop down box (You can find this information from the Bill Edwards database)

e.      Select the active database agent from the second drop down box (DBVIZ_NONPROD for the non-prod controller and DB_AGENT_717 for production)

f.        For HostName or IP Address put in any of the destination hosts from the firewall questionnaire

g.       Check the Use Service Name box

h.      In the SID or Service_Name field, put in the service name provided by the app team (from step 1)

i.         For Listener Port put in the port number from the firewall questionnaire

j.        For Username put in: app_7b3_user

k.       For password put in the password provided by network security, or if it was changed use the password it was changed to.
